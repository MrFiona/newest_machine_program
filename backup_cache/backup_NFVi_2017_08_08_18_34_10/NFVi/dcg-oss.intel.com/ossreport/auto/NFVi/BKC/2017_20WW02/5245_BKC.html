Key Updates

\xc2\xb7       \nThis is the initial NFVi release based on\nAT&T AIC project \xe2\x80\x93 Day1 configuration requirement

\xc2\xb7       \nThe release is based on Ubuntu14.04.5 with\nkernel 3.13.0-85 generic

\xc2\xb7       \nPlease refer to the HW/SW ingredient table\nbelow for detailed configuration

\xc2\xb7       \nQAT is integrated in the BKC package but it hit\nan installation issue with current configuration, no QAT validation coverage\nthis week. With the short term w/a in Boot Grub,\ninternally verified the QAT component with BKC environment\n- got positive sanity check results and will\nstart the QAT validation from next cycle

\xc2\xb7       \nTotal test cases (w/o QAT): 123;\nAttempted test cases: 123 (Attempted rate: 100%); Passed test\ncases: 109 and pass rate is 88.62% .14 test\ncases failed with total 7 related sightings. Please visit HPQC for test details

Key Issues

\xc2\xb7       \n5345808 - [NFVi_WFP][WW02] system will crash when install QAT\nacceleration

o   This QAT issue has been root caused, MPS mismatch between the\nEP and the upstream bridge, BIOS will fix the issue

o   Currently, BKC will implement a short-term w/a to unblock the\ninstallation of QAT on the platform

\xc2\xb7       \n5345841 - [NFVi _WFP][WW02][Niantic/FVL] testpmd vlan\nfilter doesn\xe2\x80\x99t work when the packets are w/o vlan ID

o   Impacted the DPDK testpmd vlan filter on Ubuntu for both\nphysical function and virtual function

Release Note

1.      This build only contain the SW package, no auto-installer\nsupport

2.      Installation step: 

o  \nInstall Ubuntu14.04.5 Server

o  \nDownload the SW package and unzip it, Copy the\nAPP folder to /opt/ of target SUT

o  \nNavigate to /opt/APP/script, run\nupdate_kernel.sh to update kernel to 3.13.0-85, reboot the system 

o  \nNavigate to /opt/APP/script, run setup.sh to\ninstall required libs and SW dependencies

Note: All SW ingredients\nare released by source code, refer to README for installation details
