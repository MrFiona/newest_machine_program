ES1\r\nAEP DIMM Setup BKM 

\xc2\xb7        Update CR FW

If AEP DIMM is not healthy, we\r\nhave two alternatives:

a.      Do\r\nAEP format

           \r\n1. Attach ITP to the SKX XDP port

           \r\n2. On the ElkValley pythonSV console, type below commands for DIMM formatting

           \r\n>>>itp.cv.resetbreak=1

           \r\n>>> sv.refresh()

           \r\n>>> fw.showBSR()

      \r\n     >>>import falconvalley.fw.fnvFwFunctions\r\nas fw

           \r\n>>>fw.dfxFormat(unlockDimm=False)

           \r\n>>>itp.cv.resetbreak=0

           \r\n3. Power cycle the system

           \r\n4. Boot to UEFI shell and update CR

              \r\nApachePassCli.efi load -f \xe2\x80\x93source  .\\< _strip.bin fw file> -dimm

b.     If step a is not working, use ITP/Jtag to update AEP FW and Syn SPD one by one,\r\nfollow the BKM Setup_TAB_AEP_BKM_v1.1.docx

 

If AEP DIMM is\r\nhealth, use UEFI shell to update all AEP FW

               ApachePassCli.efi load \xe2\x80\x93f -source\r\n<path_to_strip.bin> - dimm

 \xc2\xb7     Goal/namespace can be created under\r\nUEFI or Linux

a.          UEFI:\r\nApachePassCli.efi create -f -goal PersistentMemoryType=AppDirect

                 ApachePassCli.efi\r\nshow -a \xe2\x80\x93pool

         \r\n       ApachePassCli.efi create \xe2\x80\x93namespace \xe2\x80\x93pool\r\n<pool_Id> Type=AppDirect

b.          Linux: ixpdimm-cli create\r\n\xe2\x80\x93f \xe2\x80\x93goal persistentmemorytype=appdirect

                   ixpdimm- cli show\r\n-a \xe2\x80\x93pool
