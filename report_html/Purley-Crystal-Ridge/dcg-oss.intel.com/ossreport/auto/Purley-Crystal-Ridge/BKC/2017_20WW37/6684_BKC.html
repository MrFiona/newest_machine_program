ES1 AEP DIMM Setup BKM 

\r\n\r\n

\xc2\xb7        Update CR FW

\r\n\r\n

If AEP DIMM is not healthy, we have two alternatives:

\r\n\r\n

a.      Do\r\nAEP format

\r\n\r\n

           \r\n1. Attach ITP to the SKX XDP port

\r\n\r\n

           \r\n2. On the ElkValley pythonSV console, type below commands for DIMM formatting

\r\n\r\n

           \r\n>>>itp.cv.resetbreak=1

\r\n\r\n

           \r\n>>> sv.refresh()

\r\n\r\n

           \r\n>>> fw.showBSR()

\r\n\r\n

      \r\n     >>>import falconvalley.fw.fnvFwFunctions\r\nas fw

\r\n\r\n

           \r\n>>>fw.dfxFormat(unlockDimm=False)

\r\n\r\n

           \r\n>>>itp.cv.resetbreak=0

\r\n\r\n

           \r\n3. Power cycle the system

\r\n\r\n

           \r\n4. Boot to UEFI shell and update CR

\r\n\r\n

              \r\nApachePassCli.efi load -f \xe2\x80\x93source  .\\< _strip.bin fw file> -dimm

\r\n\r\n

b.     If\r\nstep a is not working, use ITP/Jtag to update AEP FW\r\nand Syn SPD one by one, follow the BKM Setup_TAB_AEP_BKM_v1.1.docx

\r\n\r\n

 

\r\n\r\n

If AEP DIMM is\r\nhealth, use UEFI shell to update all AEP FW

\r\n\r\n

               ApachePassCli.efi\r\nload \xe2\x80\x93f -source <path_to_strip.bin> - dimm

\r\n\r\n

 \xc2\xb7     Goal/namespace can\r\nbe created under UEFI or Linux

\r\n\r\n

a.          UEFI:\r\nApachePassCli.efi create -f -goal PersistentMemoryType=AppDirect

\r\n\r\n

                 ApachePassCli.efi\r\nshow -a \xe2\x80\x93pool

\r\n\r\n

         \r\n       ApachePassCli.efi\r\ncreate \xe2\x80\x93namespace \xe2\x80\x93pool <pool_Id> Type=AppDirect

\r\n\r\n

b.        \r\n Linux:\r\nixpdimm-cli create \xe2\x80\x93f \xe2\x80\x93goal persistentmemorytype=appdirect

\r\n\r\n

                   ixpdimm- cli show -a\r\n\xe2\x80\x93pool

\r\n\r\n

     \r\n             ixpdimm-cli create \xe2\x80\x93namespace\r\n\xe2\x80\x93pool <pool_Id> Type=AppDirect

System Scope Log

\r\n\r\n\r\n\r\n

\xc2\xb7       It is a log file that\r\nis generated by the SysScope tool from BKC systems. SysScope Tool and the user\r\nguide can be found in the BKC pkg, BKC consumers can use the SST tool on their\r\nown systems to generate an XML file describing their configuration.

\r\n\r\n

\xc2\xb7        For easy readable, we\r\ncapture the system\xe2\x80\x99s key info (CPU/PCH/BIOS/OS/AEP info) from BKC systems and\r\nput here

\r\n\r\n

\xc2\xb7 \r\n     By comparing XMLs, any differences between Intel\xe2\x80\x99s BKC system\r\nconfig and BKC consumer systems can be quickly identified (cmd sample:\r\n/SystemScope/SystemScopeCmdLine \xe2\x80\x93comparebkc <SST_BKC.xml>)
